Refdog output currently builds to working HTML under `/docs/refdog`, but MkDocs still reports many source validation warnings because the markdown is not MkDocs-native.

Main causes:

- internal markdown links use `.html` targets instead of `.md`
- some generated indexes use raw HTML anchors to `.html` pages
- some pages use absolute `/docs/refdog/...` links, which MkDocs leaves as-is
